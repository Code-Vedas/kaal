# FAQ / Troubleshooting

## Can I use it without a framework?

Yes. `kaal` is plain Ruby.

## How do I trigger one run manually?

```bash
bundle exec kaal tick
```

## How do I inspect current config and registered jobs?

```bash
bundle exec kaal status
```

## Which backend should I use?

- `memory` for local development
- `redis` for distributed lock coordination
- `sqlite` for simple SQL-backed installs
- `postgres` or `mysql` when you want advisory/named locks plus SQL persistence

## Why are jobs dispatching more than once?

Check:

- you are not using `memory` in a multi-node deployment
- `lease_ttl` is at least `window_lookback + tick_interval`
- each job key is unique
